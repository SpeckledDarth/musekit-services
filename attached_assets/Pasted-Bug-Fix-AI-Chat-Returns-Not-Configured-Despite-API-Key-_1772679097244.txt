Bug Fix: AI Chat Returns "Not Configured" Despite API Key Being Set — MEDIUM
You are working in the musekit-services Repl, which contains the @musekit/services package. This package provides backend services for a SaaS platform — AI chat, background jobs (BullMQ), notifications, and webhooks.

Stack: TypeScript, OpenAI SDK (used for all providers via baseURL), @musekit/database (Supabase client), BullMQ, Upstash Redis.

The Problem
The handleChatMessage() function always returns the fallback message "AI chat is not configured yet" even when the XAI_API_KEY environment variable is set. The function should detect the env var and use it to call the xAI API, but instead it falls through to a database lookup that fails.

Root Cause
File: src/ai/config.ts — function getAIApiKey() (lines 78-117)

The function checks process.env.XAI_API_KEY first (line 80). If the env var exists, it returns immediately. If not, it falls through to query an api_keys database table. If that table is empty or the query fails for any reason, the function throws. The caller (handleChatMessage) catches the throw and returns the "not configured" fallback.

The env var check at line 80 should short-circuit the entire flow. But at runtime, process.env.XAI_API_KEY appears to be undefined despite being configured as a secret in the hosting environment. This is the primary issue to diagnose. Additionally, the database fallback has no error handling — if the table is empty, unreachable, or has a schema mismatch, it throws an unrecoverable error with no logging, making root cause diagnosis impossible.

Current code — src/ai/config.ts (the problematic function):

export async function getAIApiKey(provider: AIConfig["provider"]): Promise<string> {
  if (provider === "xai") {
    const key = process.env.XAI_API_KEY;
    if (key) return key;             // <-- should return here, but env var appears undefined
  }
  if (provider === "openai") {
    const key = process.env.OPENAI_API_KEY;
    if (key) return key;
  }
  if (provider === "anthropic") {
    const key = process.env.ANTHROPIC_API_KEY;
    if (key) return key;
  }
  // Falls through to database lookup — table doesn't exist, throws
  const supabase = createAdminClient();
  const providerServiceMap: Record<string, string> = {
    xai: "xai",
    openai: "openai",
    anthropic: "anthropic",
  };
  const service = providerServiceMap[provider];
  if (!service) throw new Error(`Unknown AI provider: ${provider}`);
  const { data, error } = await supabase
    .from("api_keys")
    .select("key_encrypted")
    .eq("service", service)
    .single();
  if (error || !data) {
    throw new Error(
      `No API key found for provider ${provider}. Set the env var or add it to api_keys table.`
    );
  }
  return (data as { key_encrypted: string }).key_encrypted;
}

What You Need To Fix
Two changes needed:

1. Add diagnostic logging and make getAIApiKey resilient to missing database tables
File: src/ai/config.ts

The database fallback in getAIApiKey should be wrapped in try/catch so a missing api_keys table doesn't crash the function. Also add logging so issues are diagnosable from server logs.

export async function getAIApiKey(provider: AIConfig["provider"]): Promise<string> {
  const envKeys: Record<string, string | undefined> = {
    xai: process.env.XAI_API_KEY,
    openai: process.env.OPENAI_API_KEY,
    anthropic: process.env.ANTHROPIC_API_KEY,
  };
  console.log(`[getAIApiKey] Checking env var for provider "${provider}": ${!!envKeys[provider]}`);
  const envKey = envKeys[provider];
  if (envKey) return envKey;
  console.warn(`[getAIApiKey] No env var found for provider "${provider}", trying database fallback...`);
  // Env var not set — try database fallback (api_keys table)
  try {
    const supabase = createAdminClient();
    const { data, error } = await supabase
      .from("api_keys")
      .select("key_encrypted")
      .eq("service", provider)
      .single();
    if (!error && data) {
      console.log(`[getAIApiKey] Found key in database for provider "${provider}"`);
      return (data as { key_encrypted: string }).key_encrypted;
    }
    if (error) {
      console.warn(`[getAIApiKey] Database query error for provider "${provider}":`, error.message);
    }
  } catch (e) {
    console.warn(`[getAIApiKey] Database lookup failed for provider "${provider}":`, e);
  }
  throw new Error(
    `No API key found for provider "${provider}". Set the ${provider.toUpperCase()}_API_KEY env var or add it to the api_keys database table.`
  );
}
