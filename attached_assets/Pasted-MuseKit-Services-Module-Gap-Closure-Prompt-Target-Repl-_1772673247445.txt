What It Does
A server-side function that accepts a chat message (plus optional history), loads the AI config, creates the provider, calls chatCompletion(), and returns the reply string. This function is what an API route calls — it contains the business logic so the API route stays thin.

Requirements
Function Signature:

export interface ChatRequest {
  message: string;
  history?: ChatMessage[];
  systemPrompt?: string;
}
export interface ChatResponse {
  reply: string;
  provider: string;
  model: string;
}
export async function handleChatMessage(request: ChatRequest): Promise<ChatResponse>
Implementation Flow:

Call getAIConfig() to load the current AI configuration
Call createAIProvider(config) to get the provider
Build the messages array:
System message first (from request.systemPrompt or config.systemPrompt or a sensible default like "You are a helpful support assistant for a SaaS application.")
Then any history messages from request.history
Then the new user message from request.message
Call provider.chatCompletion(messages)
Return { reply, provider: config.provider, model: config.model || defaultModel }
Error Handling:

If getAIConfig() fails, use default config (provider: "xai")
If getAIApiKey() returns empty/undefined (no API key configured), return a helpful fallback:
{
  reply: "AI chat is not configured yet. Please ask an administrator to add an AI API key in the admin settings.",
  provider: "none",
  model: "none"
}
If the AI provider call fails (network error, invalid key, rate limit), catch the error and return:
{
  reply: "I'm having trouble connecting right now. Please try again in a moment, or contact support if the issue persists.",
  provider: config.provider,
  model: config.model || "unknown"
}
Log errors to console.error with context (provider, model, error message) for server-side debugging
